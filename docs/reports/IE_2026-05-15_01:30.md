# IE Execution Report
**Date:** 2026-05-15 01:30

## Objective
Buff the "Ritmo Acelerado" battle passive to grant an accumulating precision bonus (`+1 pre`, up to `+2`) upon consecutive successful hits against the same target. The bonus is lost if the attack misses or if the character does not attack the target (handled cleanly via the absence of the target's `HIT` message in the action history).

## Changes Made
1.  **Configuration Update (`data/BattlePassives.json`):**
    - Added `pre_bonus_per_hit` (default: 1) and `max_pre_bonus` (default: 2) to the `Ritmo Acelerado` parameters.
2.  **Implementation (`battle/BattlePassives.py`):**
    - Introduced state tracking (`tracked_target_id` and `current_bonus`) to the `RitmoAcelerado` class.
    - Added an `on_roll_modify` hook to check the current target, apply the precision bonus if matching, or reset the tracking state if attacking a new target.
    - Kept the existing `on_attack_end` logic for action cost reduction based on the `threshold_roll`.
    - Added an `on_turn_end` hook that uses `HistoryEmitter.hit(target_id)` to verify if the character successfully hit the tracked target. If present, the bonus is incremented; otherwise, the tracking is cleared.
3.  **Testing (`tests/battle/test_RitmoAcelerado.py`):**
    - Rewrote the test file to ensure comprehensive coverage of the new features.
    - Added `test_ritmo_acelerado_consistent_activation` to verify the original `MOVE_ACTION` conversion logic.
    - Added `test_ritmo_acelerado_pre_bonus_accumulates` to verify precision accumulation.
    - Added `test_ritmo_acelerado_miss_clears_bonus` to verify the bonus is cleared on a miss.
    - Added `test_ritmo_acelerado_different_target_clears_bonus` to verify the bonus clears when attacking a different target.
    - Added `test_ritmo_acelerado_non_attack_action_clears_bonus` to verify the bonus clears when performing a non-attack action.
    - Executed the full test suite (`pytest tests/`), resulting in 185 passing tests.
