# IE Report: [ARCH.RULES.BATTLE.ATTACK_LOAD] Compliance and Refactoring

**Date:** 2026-05-12 01:03
**Status:** Completed

## 1. Objective
Refactor the battle engine and passive abilities to strictly follow the `[ARCH.RULES.BATTLE.ATTACK_LOAD]` standard. This involves centralizing attack/defense die and stat modifications within the `AttackLoad` payload to ensure action-scoped resolution and decouple combat mechanics from entity state.

## 2. Changes Made

### Core Engine (`core/Events.py`)
- Updated `AttackLoad` data structure to include `atk_die` and `def_die` fields.
- Documented these new fields in `[ARCH.DOC.core.Events]`.

### Battle Actions (`battle/BattleActions.py`)
- Refactored `AttackAction.execute` to:
    - Initialize `atk_die` and `def_die` in `AttackLoad` from character defaults.
    - Use `attack_load.atk_die` for attack rolls (including Master Rolls in AoE).
    - Use `attack_load.def_die` for defense rolls.
    - Calculate hit success using action-scoped `pre` and `grd` from `AttackLoad`.
- Updated hook builders:
    - `_build_swap_atk_def_die`: Now modifies `attack_load.atk_die` directly instead of the character's base stat.
- Created comprehensive documentation in `[ARCH.DOC.battle.BattleActions]`.

### Battle Passives (`battle/BattlePassives.py`)
- Refactored multiple passives to modify `AttackLoad` properties instead of applying `EphemeralModifier` to characters:
    - **PosturaDefensiva**: Now modifies `attack_load.pre` in `penalty_hook`.
    - **Bloquear**: Now modifies `attack_load.bda` in `bonus_contra_ataque_hook`.
    - **PosturaBatalha**: Reroll logic now uses `attack_load.def_die` and updates `attack_load.defense_roll` and `attack_load.gda` directly.
- Updated documentation in `[ARCH.DOC.battle.BattlePassives]`.

### Testing (`tests/`)
- Updated `tests/battle/test_GolpeDeEscudo.py` and `tests/battle/test_PosturaDefensiva.py` to validate `AttackLoad` property mutations.
- Verified that all 146 project tests pass.
