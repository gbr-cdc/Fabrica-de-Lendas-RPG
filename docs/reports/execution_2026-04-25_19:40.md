# Execution Report: Unified Reference Management
**Date:** 2026-04-25
**Time:** 19:40
**Mission:** Consolidation of Documentation Utilities

## 1. Overview
This report documents the consolidation of `gdd_reader.py` and `ref_extractor.py` into a single, more powerful utility: `ref_manager.py`. The goal was to provide a unified interface for extracting documentation sections based on tags while implementing robust dependency resolution across multiple files.

## 2. Changes Made
- **Created `utilities/ref_manager.py`**: A unified script that merges and improves upon the legacy extraction tools.
- **Enhanced Dependency Resolution**: Ported the `[DEPENDS: ...]` logic from the GDD reader to work globally across all documentation files (`architecture.md`, `docs/GDD/`, etc.).
- **Implemented Cross-File Tracking**: Added logic to detect when a dependency originates from a different file than its parent, automatically prepending a `> [FROM: filename]` header.
- **Improved Extraction Logic**: 
    - Implemented header-level awareness to capture entire markdown sections correctly.
    - Added automatic stripping of trailing horizontal rules (`---`) to avoid duplicate separators in joined outputs.

## 3. `ref_manager.py` Functionalities

### 3.1 Targeted Extraction
The script uses `[TAG]` identifiers to locate content. If a tag is found in a header, it captures the entire section until a header of equal or higher level is encountered. This ensures that tables, lists, and sub-points belonging to a section are extracted together.

### 3.2 Recursive Dependency Resolution
If an extracted section contains a `[DEPENDS: TAG1, TAG2]` line, `ref_manager` will:
1. Locate the dependency tags within the search path.
2. Recursively extract those dependencies (and their own dependencies).
3. Prepend the resolved dependencies to the final output, separated by horizontal rules.

### 3.3 Cross-File Origin Information
To facilitate navigation and context, any dependency found in a file different from the one containing the parent tag is marked with a source indicator:
`> [FROM: 00_Base_Rules.md]`

### 3.4 Command Line Interface
The script supports flexible input:
- **Explicit Path**: `python3 utilities/ref_manager.py <file_or_dir> [TAG1] [TAG2]`
- **Smart Guessing**: `python3 utilities/ref_manager.py [TAG]`
    - If the tag contains `ARCH`, it defaults the search to `architecture.md`.
    - Otherwise, it defaults to searching the `docs/GDD/` directory.

## 4. Status
The legacy scripts `gdd_reader.py` and `ref_extractor.py` are now redundant and can be safely removed or archived once the transition is verified in all workflows.
