# IE Report — Defensive Reaction Flow Refinement
**Date:** 2026-05-12 23:12

## Summary
Corrected the combat signal ordering so that `on_defense_reaction` fires **only after a hit is confirmed** and damage logic is **fully gated** behind the post-reaction hit state.

---

## Changes

### `battle/BattleActions.py` — `AttackAction.execute`
- **Before:** `on_defense_reaction` was emitted unconditionally (even on misses), before the hit/miss branch.
- **After:** Hit is determined first. If `GdA > (grd - pre)`, `attack_load.hit = True` is set and `on_defense_reaction` is emitted. After all reactions, hit is **re-evaluated**: if `GdA <= (grd - pre)`, `attack_load.hit = False`. All damage logic and `on_hit_check` are gated behind the final `attack_load.hit` state.
- `on_hit_check` now always fires unconditionally after HIT/MISS record (post-reaction).

### `battle/BattlePassives.py` — Defensive Reaction Hooks
- **`GracaDoDuelista.reacao_evasao_hook`**: Removed redundant GdA threshold re-check. The hook relies on the engine guarantee that it only fires on confirmed hits.
- **`Bloquear.reacao_bloqueio_hook`**: Same simplification. Changed counter trigger from `< counter_threshold` to `<= counter_threshold`.
- **`PosturaBatalha.on_defense_reaction`**: Same simplification.

### `data/BattlePassives.json` — Bloquear template
- Changed `counter_threshold` from `-3` to `0`. The counter now triggers when the block completely nullifies the attack (`GdA <= 0`), which is reachable with the `reduction_die: 4` and makes mechanical sense ("complete block → counter opportunity").

### `tests/battle/test_Bloquear.py` — `test_bloquear_counter_bonus`
- Rewrote test to use correct dice scheduling: `atk=6, def=5 → GdA=1` (confirmed hit), block roll `1 → GdA=0 <= counter_threshold`, triggering the counter.
- Added intermediate assertion that the counter is set on the passive before the counter-attack occurs.
- All template values retrieved dynamically per `[ARCH.TEST_QUALITY]`.

### `architecture.md` (via ref_manager)
- Updated `[ARCH.DOC.battle.BattleActions.AttackAction.execute]` to reflect the new signal sequence.
- Updated `[ARCH.DOC.battle.BattlePassives.GracaDoDuelista.get_hooks]` — corrected signal name, removed manual GdA check from description.
- Updated `[ARCH.DOC.battle.BattlePassives.Bloquear.get_hooks]` — corrected signal name, updated counter threshold semantics (`<=`).
- Updated `[ARCH.DOC.battle.BattlePassives.PosturaBatalha.get_hooks]` — removed manual GdA check from description.
