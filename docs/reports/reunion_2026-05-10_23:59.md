# Reunion Report: Agent Reference Manager Skill Updates
Date: 2026-05-10 23:59

## Problems Identified
- The AI interface truncates large outputs from the `ref_manager` terminal responses, making it difficult for agents to process dependencies or read large files.
- The `ref_manager` dependency resolution depth is 1 by default, but agents were sometimes unsure when to use the `--depth` flag.
- Agents needed to be aware of the `output.txt` mitigation strategy to avoid truncation and to confidently fetch multiple tags in a single call.

## Changes Made
- Updated `[AGENT.REF_MANAGER]` documentation in `agent_rules.md` to:
  - Add explicit instructions about `output.txt` being used to mitigate truncation.
  - Clarify that `--depth 1` is the default and agents should only use the `--depth` flag if explicitly requested.
  - Incentivize agents to fetch multiple tags per call now that `output.txt` reliably handles large outputs.
