# Report: Data-Driven Passives Implementation (IE_2026-05-12_05:15)

## Summary of Implemented Features
The `BattlePassive` system has been refactored to support dynamic parameterization via JSON templates. This decoupling allows game designers to adjust ability balance (thresholds, dice sizes, costs) without modifying Python code.

### 1. Structural Changes
- Added `BattlePassiveTemplate` dataclass to `core/Structs.py`.
- Updated `BattlePassive` base class constructor in `core/BaseClasses.py` to store a template reference.

### 2. Data Management
- Created `data/BattlePassives.json` with configuration for all current passives.
- Updated `DataManager.py` with `load_passive_templates` and `get_passive_template` methods.
- Updated `BattleManager.py` to inject templates into passives during character initialization.

### 3. Passive Refactoring
- All passives in `battle/BattlePassives.py` now use `self.template.parameters.get()` for their logic.
- Hardcoded constants have been eliminated in favor of data-driven variables.

### 4. Test Improvements
- Updated `tests/utils/test_utils.py` to load passive data in all battle tests.
- Refactored `tests/battle/test_RitmoAcelerado.py` and `tests/battle/test_GracaDoDuelista.py` to use dynamic parameters for assertions, ensuring they remain valid after balance changes.

## Known Limitations & Next Steps
- **Status Effects**: Currently, only `BattlePassives` are fully data-driven. `StatusEffect` logic (which inherits from `BattlePassive`) could also be migrated to templates in a future iteration to standardize duration and intensity parameters.
- **Validation**: Added basic error handling for missing templates, but a more robust validation schema (e.g., JSON Schema) for `BattlePassives.json` could be implemented to prevent runtime errors from malformed data.
