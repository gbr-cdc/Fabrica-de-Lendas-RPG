## 1. Executive Summary
The current test suite for `DataManager` is heavily dependent on specific game data (values within JSON files). This creates a brittle testing environment where balance changes or content updates break logic tests. The goal is to refactor these tests to focus on **behavior, integrity, and schema**, ensuring the loading pipeline is robust regardless of the data content.

## 2. Discussion Overview
We identified that `tests/core/test_DataManager.py` currently asserts specific attributes of characters and combat styles (e.g., asserting that "Destruidor" has FIS as its main stat). 

### Identified Issues:
- **Brittle Assertions**: Tests fail if JSON values change, even if the code is correct.
- **Maintenance Burden**: Every time the game balance is adjusted, developers must update unit tests.
- **Poor Focus**: Unit tests should verify that the `DataManager` *can* load a character, not *what* the character's stats are.

## 3. Proposed Suggestions
To align with the **Test Quality Standards**, the following changes are proposed:

### 3.1. Registry Population Verification
Instead of checking for a specific ID, verify that the internal registries (`_combat_styles`, `_characters`, etc.) are not empty after the loading methods are called.
```python
dm.load_combat_styles('data/CombatStyles.json')
assert len(dm._combat_styles) > 0
```

### 3.2. Type & Enum Integrity
Verify that the loaded data has been correctly hydrated into class instances and that string values from JSON were correctly converted into Enums.
- Check `isinstance(obj, ExpectedClass)`.
- Check `isinstance(obj.stat, AttributeType)`.

### 3.3. Dynamic Key Validation
Use the proposed reading utility (see Section 4) to iterate over all IDs present in the JSON and ensure `DataManager.get_*` can retrieve them all successfully.

## 4. Proposed Reading Utility: `JsonIntegrityChecker`
I suggest creating a helper utility within the test suite (or a standalone script in `utilities/`) with the following responsibilities:

### Features:
1. **Key Extraction**: Automatically reads the top-level keys from a JSON file to provide a list of valid IDs to the test runner.
2. **Schema Sanitization**: Performs a "dry run" to check if required fields (like `name`, `atq_die`, `CombatStyle`) are present in the JSON before hydration.
3. **Dynamic Test Injection**: Allows pytest to parametrize tests based on the actual content of the `data/` directory, catching corrupted files immediately without manual test updates.

### Workflow Example:
```python
def test_all_characters_load_successfully():
    keys = get_json_keys('data/Characters.json')
    for key in keys:
        char = dm.get_character(key)
        assert char.char_id == key
        # Verify hydration
        assert char.combat_style is not None
```

## 5. Conclusion
By implementing these changes, we decouple the engine logic from the game content. The `DataManager` tests will now serve as a "gatekeeper" that catches both code bugs in the loading logic AND corruption in the JSON data format, without being sensitive to balance tweaks.
