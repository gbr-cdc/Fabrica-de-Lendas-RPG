# Execution Report: Fix PvP Simulator Drawing Every Battle
**Date:** 2026-05-07 00:55
**Workflow:** [WORKFLOWS.INDEPENDENT_EXECUTION]

## Overview
Identified and fixed a bug where all PvP simulations in multi-battle mode were ending in a draw. The issue was caused by the `BattleResult.winners` and `BattleResult.losers` lists not being populated by the `BattleManager`, which led the summary view to default to a draw.

## Changes Implemented

### Logic
- **`core/BaseClasses.py`**: 
    - Added `IBattleJudge` Protocol to formalize the judge interface.
    - Added `get_graveyard()` to `IBattleContext`.
- **`battle/Judges.py`**:
    - Updated `BattleJudge.rule()` to accept a `BattleResult` object.
    - Implemented logic to populate `winners` and `losers` based on team outcomes when a terminal state is reached.
    - Integrated `get_graveyard()` to ensure all participants (including dead ones) are accounted for in the results.
- **`battle/BattleManager.py`**:
    - Implemented `get_graveyard()` to expose dead characters.
    - Updated all calls to `judge.rule()` to pass `self.battle_result`.

### Tests
- **`tests/battle/test_Judges.py`**: Updated to verify `winners` and `losers` population.
- **`tests/battle/test_EngineEdgeCases.py`**: Updated to match new `rule()` signature and verify terminal state population.

### Documentation
- Updated `[ARCH.DOC.core.BaseClasses]`, `[ARCH.DOC.battle.Judges]`, and `[ARCH.DOC.battle.BattleManager]` in `architecture.md` to reflect the new APIs and responsibilities.

## Verification
- **Automated Tests**: All 21 tests passed (`python3 -m pytest`).
- **Mono Simulation**: Verified that character HP stats are displayed at the end of the battle.
- **Multi Simulation**: Verified that victories and defeats are correctly aggregated across 10000 battles.

## Conclusion
The PvP simulator now correctly reports battle outcomes. The architectural decision to delegate result population to the Judge maintains a clean separation of concerns, as the Judge is the component that understands victory conditions.
