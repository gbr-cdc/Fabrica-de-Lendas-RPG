# Implementation Report: Ritmo Acelerado
**Date:** 2026-05-11 21:10

## 1. Objective
Implement the "Ritmo Acelerado" passive ability for the "Retalhador" combat style and enhance the battle engine's handling of action-scoped stat modifications.

## 2. Changes Performed

### 2.1 Architectural Enhancements
- **Stat Scoping in Payloads (`core/Events.py`)**: Added `pre`, `bda`, `bdd`, and `grd` attributes to the `AttackLoad` dataclass. This allows abilities to modify combat stats specifically for one resolution step without affecting the character's base/modified state.
- **Event History (`core/Events.py`)**: Expanded `HistoryEmitter` with `passive(name, id)` and `atk_load(attr, delta, current)` methods for better debugging and testability.
- **Action Contextualization (`battle/BattleManager.py`)**: Added `self.current_action` to `BattleManager`, allowing passive hooks to inspect and modify the properties of the action that triggered them (e.g., changing its `BattleActionType`).
- **Protocol Updates (`core/BaseClasses.py`)**: Defined `IActionStateContext` and integrated it into `IPassiveContext` and `IBattleContext`.

### 2.2 Battle Engine Logic
- **`AttackAction` Refactor (`battle/BattleActions.py`)**: Updated the combat flow to populate `AttackLoad` with attacker and defender stats at the start of resolution. The hit calculation formula now uses the values stored in the payload.

### 2.3 Passive Implementation
- **`RitmoAcelerado` (`battle/BattlePassives.py`)**:
    - **Rhythm Tracking**: Uses a counter to track consecutive rolls of 7+.
    - **Action Economy**: Converts successful attacks into `MOVE_ACTION` (reducing tick cost).
    - **Precision Bonus**: Grants a +2 `PRE` bonus on the third consecutive attack.
    - **Double-Count Prevention**: Implemented `processed_actions` tracking to ensure multi-target or AoE attacks only progress the rhythm once per action.

## 3. Verification Results
- **Integration Tests**: Created `tests/battle/test_RitmoAcelerado.py`.
    - Verified `MOVE_ACTION` conversion on 7+ rolls.
    - Verified +2 `PRE` application on 3rd attack.
    - Verified rhythm reset on rolls < 7.
- **Full Suite**: Ran `pytest`.
    - **Results**: 151 passed.
    - **Status**: System integrity maintained.
