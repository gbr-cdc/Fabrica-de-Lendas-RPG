# Reunion Report: Workflow Evolution and Test Quality Standards

**Date:** 2026-04-23 19:10
**Context:** [REUNION]
**Participants:** Senior Architect (Agent), USER

## 1. Problems Identified
*   **Low-Quality TDD**: Agents were prioritizing 100% coverage over meaningful logic verification, leading to excessive mocking.
*   **Implementation Coupling**: Tests were too dependent on internal data structures (e.g., timeline indices), making them brittle during refactors.
*   **Context Overload**: The [PLANNING] context was becoming bloated with test implementation details.

## 2. Changes Implemented

### 2.1 Workflow Refactor ([execution.md](../../docs/contexts/execution.md))
*   **Two-Phase Execution**: Mandatory split between **RED** (Test Scaffolding) and **GREEN** (Logic Implementation).
*   **Integration Focus**: Encouraged the use of real entities (`Character`, `StatModifier`) over `MagicMock`.

### 2.2 Planning Context ([planning.md](../../docs/contexts/planning.md))
*   **Acceptance Criteria (AC)**: Added as a mandatory section for every mission.
*   **Behavioral Specs**: Planning now focuses on defining "What" to test (Scenarios) rather than writing test code, reducing token bloat and context overload.

### 2.3 Global Rules ([agent_rules.md](../../agent_rules.md))

*   Added **Section 4: Quality Guardrails (QG)** to establish a high-level commitment to Behavior-Driven Tests.

## 3. Test Quality Standards
Tests must now adhere to the following principles:
*   **Behavior over Implementation**: Verify "What" happened (final state, logs) rather than "How" it happened internally.
*   **Controlled Mocking**: Real instances are preferred for core engine logic. Mocks are reserved for external controllers.
*   **Decoupling**: Public API stability is prioritized to allow internal engine refactors without breaking the test suite.

## 4. Future Roadmap: Semantic Invariants
A future [PLANNING] session will be dedicated to defining **Semantic Invariants** (e.g., HP/MP bounds, Timeline sorting consistency). These will be formalized in `architecture.md` to ensure the engine's mathematical integrity across all implementations.
