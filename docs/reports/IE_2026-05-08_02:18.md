# Test Quality Audit Report: Battle Module

## Overview
An audit of the `tests/battle` directory was conducted on 2026-05-08 to verify adherence to `[ARCH.TEST_QUALITY]` standards and evaluate the organization of the test suite.

## Findings: Quality Standards

| Rule | Status | Observation |
| :--- | :--- | :--- |
| **Behavior over Implementation** | ⚠️ Partial | Newer tests (`test_GolpeDeEscudo.py`, `test_BattlePassives.py`) are behavioral. Older ones (`test_multi_target.py`) mock internals and private state. |
| **Controlled Mocking** | ❌ Divergent | Excessive use of `MagicMock` for `IBattleContext`. Should transition to `BattleTestContext`. `DiceManager` is used but sometimes bypassed by manual mocks in older tests. |
| **Entity Factory** | ⚠️ Partial | Generally followed, but completely ignored in `test_EngineEdgeCases.py` and `test_multi_target.py`. |
| **Data Integrity** | ❌ Divergent | `tests.utils.json_integrity_checker` is not used. Integration tests are not iterating over all game data IDs. |
| **Lifecycle Auditing** | ❌ Divergent | No tests verify the `EventBus` subscriber baseline. Ephemeral hooks are not being audited for cleanup. |
| **Battle Context** | ❌ Divergent | `BattleTestContext` is rarely used; `MagicMock` is the default, making tests fragile to context API changes. |
| **Structured History** | ✅ Good | Most tests correctly assert against `TAG|...` formatted strings. |
| **AttackAction Data Loading**| ❌ Divergent | `AttackActionTemplate` is manually instantiated in almost all tests except `test_GolpeDeEscudo.py`. This risks desync with `data/AttackActions.json`. |

## Findings: Organization
1. **Redundancy & Fragmentation:** `test_area_attacks.py` and `test_multi_target.py` contain logic that should be unified under `test_BattleActions.py` or grouped into a `functional/` subfolder if they are integration-focused.
2. **Mocking Anti-patterns:** `test_multi_target.py` and `test_EngineEdgeCases.py` represent "legacy" mocking styles that are highly coupled to the implementation and should be refactored or removed in favor of `entity_factory`.
3. **Consistency:** There is a clear "quality gap" between tests created during recent missions (e.g., `test_GolpeDeEscudo.py`) and earlier tests.

## Recommendations
1. **Refactor Action Tests:** Update `test_BattleActions.py` and consolidated action tests to use `DataManager` for template loading.
2. **Standardize Context:** Replace `MagicMock` with `BattleTestContext` in behavioral tests.
3. **Implement Lifecycle Audits:** Add `assert context.event_bus.subscriber_count == baseline` to tests involving ephemeral hooks (like `PosturaDefensiva` and `StatusEffects`).
4. **Consolidate Files:** Merge `test_multi_target.py` and `test_area_attacks.py` into a more robust integration test suite for actions.
5. **Enforce Entity Factory:** Refactor `test_EngineEdgeCases.py` to use `create_dummy_character`.
