# Reunion Report: 2026-04-29 12:40

## Problems Identified
1. **Ambiguous "Smart Placement" Documentation:** The previous rules did not explain the hierarchical logic (siblings then parent) used by `ref_manager`, leading to confusion when creation failed.
2. **Incorrect Create Usage:** Agents were passing parent tags to the `--create` command instead of the full target tag, causing structural errors.
3. **Shell Expansion Issues:** Lack of enforcement for single quotes in CLI commands led to potential shell interpretation of content strings.
4. **Duplicate Tags:** No explicit rule against creating already existing tags.

## Changes Made
### [AGENT.REF_MANAGER] in agent_rules.md
- **Smart Placement Explanation:** Added detailed logic (last sibling -> parent -> fail).
- **Target Tag Enforcement:** Explicitly mandated passing the full NEW tag to `--create`.
- **Shell Safety:** Mandated the use of single quotes `' '` for content arguments.
- **Uniqueness Check:** Added a constraint that target tags for creation must not already exist.
- **General Cleanup:** Refined the descriptions of update and delete commands for consistency.
